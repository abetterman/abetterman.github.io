ES2015 Scope & Hoisting -let & const

ES2015 介绍了两个声明变量的新方法：let 和 const .

let

let 与 var 的不同：

  • not hoisted
  • block-scoped

Not Hoisted

如前所述，由 var 声明的变量会获得提升，而由 let 声明的变量却不会获得提升，在声明前引用，会导致引用错误：

let 声明的变量没有提升，也省了很多麻烦，变得很简单了。

Block-Scoped

var 是函数作用域的，即只有在一个函数内部，它才是局部作用域的。

而 let 是块级作用域的，这是 JavaScript 中的一个新的作用域机制，作用域不再限于函数和全局。

任一对花刮号 {} 都会建立一个新的作用域，在其中用 let 声明的变量只在这个作用域内可用。

同样，在循环语句 for 和 while 中，用 let 声明的变量，也只在循环中可用。而 var 声明的变量却在循环之外仍然可用。

在循环语句中的条件 let 声明的变量，也是块级作用域，只在循环中可用，即使他们不在花刮号内。

另外，用 let 重复声明一个变量会失败，而用 var 则不会：

const

No reassignment

所有适用于 let 的新规则同样也适用于 const, 此外，还有一些增加的规则：1. 用 const 声明的变量不能被重新赋值。

No empty initializations

const 不允许使用空初始化。而使用 var x 或者 let x ，将得到 undefined 值，const x 却会抛出一个错误。它必须在声明时给定一个具体的值。

Usage

事实是 var 已经过时了，从现在开始，应该使用 let 和 const。

Which One?

Preferring const

关于 let 和 const 有两种思想流派，首先是尽可能使用 const， 你会发现需要重新赋值的变量比常量少得多。

努力使用 const，只在必要时回退到 let。

Preferring let

其他人，包括 Kyle Simpson，更喜欢用 let 而不是 const，这个想法是 const 应该为 true 常量、数学常量（比如 PI）保留。

也许第一种策略更好，无论哪里都尽可能地使用 const ，只在需要重新赋值时降级为 let。

Introduction to Callbacks

JavaScript 最强大的属性之一就是函数是一等对象。这意味着它们与任何其他对象一样，并且具有与标准对象相同的属性。实际上，我们应该将函数视为可调用的对象。

像一个对象一样，我们可以在函数上存储属性。

一个对象可用任何类型数据作为属性，函数也是如此，甚至可以将一个函数设为另一个函数的属性：

一个函数被传递给另一个函数作为参数，或者从另一个函数返回，都是完全合法的。

函数作为参数：

这就是回调的定义：回调是一种传递给另一个函数的函数。

传递函数作为参数和返回函数允许我们：

  • 消除代码重复
  • 允许异步执行
  • 允许事件绑定，比如用户交互
  • 隐藏数据并创建一个令人愉快的用户界面
  • 防止作用域污染

Closures

闭包允许我们动态创建函数，实现封装，并创建接口以与我们的代码进行交互。

Scope Persistence

函数创建时，它会保留对其创建作用域内的访问。如果在另一个函数内创建，即使在该外部函数返回之后，它也会保留对外部函数作用域内的访问。

这样，就创建了一个闭包。

即使 fnGenerator 已经结束了执行，但其作用域仍然保持在内存中，返回的函数 fnReturned 仍然可以访问这个作用域内的变量。这个概念，我们用来解决前面的代码重复的问题。

MDN 定义：

The combination of a function and the lexical environment within which that function was declared.

The Solution

利用闭包特性，我们可以写出 generator functions 或者 factory functions。

生成器/工厂函数基于传递给它的参数来创建和返回函数的。

add10 保持访问 10 并且与任何传递给它的数值相加。同样， add20 和 add30 分别保持对 20 和 30 的访问。

如此，就摆脱了代码的重复。

Encapsulation

凭借这个作用域持久的想法，我们可以编写向用户隐藏数据的函数，这被称作为封装。如此，我们就可以避免用户无意间修改了数据的值。

假设我们编写一个函数，可以跟踪它被调用的次数，并且不希望用户可以修改这个值。

用户只能与 incrementCounter 函数交互，counterGenerator 函数被隐藏，与 counter 变量交互的唯一方式是调用incrementCounter 函数。这类似于汽车驾驶员无需关心引擎一样。软件更新，只需更新 code base，而无需更新用户接口，用户的操作方式无需任何更改。这就是封装的好处。
