Variable Scope

Global Scope

x,y,fn 都是 Global Scope

Local Scope

x,y,fn 都是 Global Scope，insideFnScope 是 Local Scope

Variable lookup mechanism: 当我们试图访问一个变量，JavaScript 引擎首先在本级作用域查找，如果查询不到，会向其父级作用域查询，或继续向上级作用域查询，直到查询到这个变量。若达到了 Global Scope 也没查询到，则会报 reference error.

Changing the Outer Scope

省略了 var ,引擎便不会在函数 fn 中创建 x 而是尝试去找到 x，同样的，引擎也尝试找到 y, 在 local scope 和 global scope 都没找到，于是，引擎就在 global scope 创建了变量 y

Best Practices

Setting new variables in the global scope is bad practice. It leads to confusion and bugs that are very difficult to find. It pollutes the global scope.

上述结果，是因为在程序运行前，JavaScript 引擎改变了代码结构：变量声明被提升到了作用域的顶部。

注意，只是变量声明被提升，变量赋值不会被提升。声明却没有赋值的变量，则自动获得值undefined

Hoisting and var

没有使用 var 声明的变量，不会获得提升。
